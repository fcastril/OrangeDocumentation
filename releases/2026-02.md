# Release 2026-02

### ✨ Nuevas Funcionalidades
![Static Badge](https://img.shields.io/badge/Module-Inventarios-green)

**Copiar Referencias:**

Ahora puedes duplicar referencias existentes de forma rápida y sencilla:

* **Botón de Copia:** En el listado de referencias aparece un botón azul con ícono de copiar (solo para referencias tipo 'R').
* **Proceso Simplificado:** Al hacer clic, se abre una ventana donde solo debes ingresar:
  - El nuevo código para la referencia
  - El nuevo nombre/descripción
* **Copia Completa:** El sistema duplica automáticamente:
  - Todos los atributos configurados
  - Materiales asociados
  - Proveedores
  - Consumos
  - Operaciones
  - Consumos por atributos
* **Acceso Directo:** Al finalizar la copia, el sistema te lleva directamente a editar la nueva referencia creada.
* **Validación en Tiempo Real:** El sistema valida automáticamente que el código ingresado no esté duplicado mientras escribes.

Esta funcionalidad es ideal cuando necesitas crear referencias similares, ahorrando tiempo al evitar configurar todo desde cero.

### 🐞 Bugs Corregidos
![Static Badge](https://img.shields.io/badge/Module-Múltiples-blueviolet)

**Corrección de Modal de Referencias (Atributos):**

* **Error:** Cuando se cerraba la ventana modal de Referencias, el indicador de carga (rueda giratoria) se quedaba visible. Además, si se volvía a abrir la ventana, el indicador seguía pegado y la ventana no respondía correctamente.
* **Afecta a:** Pedidos, Traslados, Entradas-Salidas, Ventas
* **Solución:** Se mejoró el funcionamiento de la ventana modal para que el indicador de carga desaparezca correctamente, y la ventana funcione sin problemas cada vez que se abre.

![Static Badge](https://img.shields.io/badge/Module-Administrativos-yellow)

**Tipo de Documento 44 en Creación de Terceros:**

* **Error:** Al crear un tercero y seleccionar tipo de documento 44, los campos de nombres y apellidos no cambiaban por "Razón Social", manteniendo un comportamiento inconsistente.
* **Causa:** La lógica de validación del formulario solo verificaba el tipo de documento 31 (NIT), sin considerar el tipo 44.
* **Solución:** Se actualizó la lógica en:
  - **JavaScript (`terceros.js`):** Función `CambioDocumento()` ahora valida ambos tipos (31 y 44)
  - **Backend (`TercerosController.cs`):** Validaciones de campos opcionales consideran ahora el tipo 44
  - **Importación (`ImportacionesController.cs`):** Proceso de carga masiva trata correctamente el tipo 44
* **Resultado:** Al seleccionar tipo 44, el sistema ahora:
  - Cambia el label a "Razón Social"
  - Oculta campos de Nombre2, Apellidos y Género
  - Realiza validaciones correctas al guardar
  - Mantiene consistencia en importación de datos

**Reporte de Pedidos - Organización por Referencia:**

* **Error:** El reporte de Pedidos por Referencia no funcionaba. El sistema solo permitía generar reportes por Consecutivo y por Cliente.
* **Causa:** Discrepancia en el nombre del archivo JavaScript (`reportePedidosReferencias.js` en lugar de `reportePedidosReferencia.js`), lo que impedía que se cargara el script necesario.
* **Solución:** Se corrigió el nombre del archivo JavaScript para que coincida con la referencia en la vista. Ahora es posible generar reportes organizados por:
  - **General:** Pedidos con desglose de líneas
  - **Por Cliente:** Pedidos agrupados por cliente
  - **Por Referencia:** Pedidos agrupados por referencia (productos) ✅ FUNCIONANDO
